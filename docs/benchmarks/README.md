# Benchmark results

A committed snapshot of the headline benchmark — eager vs vLLM — so the numbers
and chart live in the repo (raw sweep outputs under `results/` are gitignored).

## Setup
- **Model:** TinyLlama/TinyLlama-1.1B-Chat-v1.0 (fp16)
- **GPU:** free Google Colab T4 (16 GB)
- **Workload:** sequence length 128, 128 output tokens, greedy
- **Backends:** `eager` (HuggingFace `.generate`, the baseline) vs `vllm`
  (continuous batching + chunked prefill)

## Files
- `tinyllama_t4_seq128.csv` — throughput + per-token latency (TPOT) by batch size.
- `backend_comparison.png` — the two-panel chart, generated by
  `scripts/plot_benchmark.py` from the CSV.

Regenerate the chart with:

```bash
python scripts/plot_benchmark.py
```

## Headline
- vLLM delivers **~2.6–3.2×** the throughput of eager across batch sizes
  (avg ~2.8×), peaking at **~1159 tok/s** at batch 16.
- vLLM holds per-token latency around **12 ms** vs eager's **33–42 ms**
  (**~65% lower**), because its continuous batching keeps the GPU busy.

## Notes
- vLLM's EngineCore hits a memory ceiling on a T4 above batch 16; the sweep is
  crash-resilient and saves every completed grid point.
- These are small-model / free-GPU numbers; the same pipeline scales to LLaMA 3
  8B on an A100 (config + notebook tier included), where absolute gains are larger.
- Run-to-run variance on Colab is roughly +/-15% (warmup, caching, shared host).
